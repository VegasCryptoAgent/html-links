Settings
Bring people in

Invite-only. Two ways in.

Production runs with REGISTRATION_MODE=invite_only. Nobody can self-register a new workspace unless an operator deliberately opens registration — which you should not do.

1

Universal join link

In-app path: Team → Universal join link. Copy the link and share it. Anyone with it creates an account and joins this workspace as the configured role (default participant). Rotate or disable the link if it leaks.

https://royaltygrid.com/?join=<CODE>

2

Email invitations

In-app path: Team → Send invitation. One-time link of the form /?invite=<token> for a specific email. Expires in 7 days. Prefer this when you need a known address and a chosen staff role (admin, accountant, manager…).

3

They land in the right place

Staff see the full ops product for their role. Participants land in the rights portal — their earnings, splits, statements, and support only. Link their user to a People record so portal data binds correctly.

Production default: invite-only. New workspaces cannot self-register. Users enter only via owner/staff invite or the universal join link.

Roles

Who can see what

Grant staff roles deliberately. Participants never see the full ledger.

Owner

Full control and ownership transfer. Everything in the workspace.

Admin

Team, settings, enterprise, ops — everything except owner-only transfer.

Accountant

Revenue, imports, royalty runs, statements, payables, plus staff reads.

Manager

Catalog, splits, people, connections, and staff reads.

Participant

Rights portal only — own earnings, splits, statements, support.

Viewer

Minimal portal access under the same isolation model.

What's inside

Every major screen, one real job

Daily

Ledger

What: Live totals — gross revenue, publishing/master pools, balances by person, warnings, audit activity.

How: Open after imports or revenue entry. Drill warnings into incomplete split sheets.

Why: One screen answers who is owed what before period close.

Core

Revenue

What: Source lines (manual or import) in exact cents.

How: Enter or import CSV/XLSX/XLS; preview; commit; reverse mistakes instead of erasing history.

Why: Allocations need immutable, provenance-tracked source rows.

Core

Catalog & People

What: Songs, releases, recordings, compositions; payees with roles, IPI, tax status.

How: Lock splits only when master + publishing each sum to 100% and reviews (if any) are done.

Why: Split versions are the law of allocation for each revenue date.

Review

Reviews

What: Participant approval of draft split sheets; disputes with evidence.

How: Manager requests review → approve / request changes / dispute from Reviews or email.

Why: Shared ownership without email chaos.

Insight

Analytics

What: Trends, top songs/people/sources, YouTube when connected, outliers.

How: Open after data lands; watch sync freshness.

Why: Spot concentration risk and import anomalies early.

Core

Contracts · Runs · Payables

What: Versioned deals, calculation runs with traces, payable export batches.

How: Draft → activate → run period → inspect trace → approve → export CSV → pay offline → confirm paid.

Why: Auditable contract math; payments stay outside the product.

Core

Statements

What: Per-person allocation statements (JSON + PDF), schedules, email jobs.

How: Deliver on demand or monthly/quarterly from the same allocation engine.

Why: Formal evidence without spreadsheet mail merges.

Portal

Rights portal

What: Participant home — own earnings, splits, contracts, statements, support.

How: Participants land here by default; staff can preview the holder experience.

Why: Transparency without exposing the whole label’s ledger.

Ops

Connections & Operations

What: YouTube via PostProxy, SFTP/IMAP, job health, email delivery, audit feed.

How: Admin attaches sources; Operations shows dead or delayed jobs.

Why: Real provenance; catch stuck work before artists do.

Setup

Enterprise & Settings

What: Private files, API keys (rg_live_…), webhooks, SSO/SCIM, IP allowlist, MFA, sessions.

How: Configure only with real credentials — readiness never invents “connected.”

Why: Label-grade controls without demo fiction.

Core workflows

What to run every week

Six paths operators actually use: invite, join, splits, import, period close, statements.

A

Invite a collaborator (email)

Team → Send invitation with email + role. They open /?invite=…, set a password, and land in the workspace. Optionally link them to a People record.

B

Share the universal join link

Team → Universal join link → Copy. Share https://royaltygrid.com/?join=…. Joiner picks name, email, password and joins as the configured role. Rotate if the link was posted too widely.

C

Split a song and lock it

Catalog → create song → master + publishing rows each = 100% → optional review → lock. Later unlocks create a new draft; historical revenue keeps the old version.

D

Import a distributor statement

Revenue → upload CSV/XLSX/XLS → preview mapping → commit → resolve unmatched → verify in Ledger and Analytics.

E

Period close

Reconcile expected totals → close the period → run royalty calculations if on the contract engine → approve payables → export batch CSV → pay offline → confirm paid.

F

Deliver statements

Statements → person + period → PDF or email. Check Operations for job status and email events.
